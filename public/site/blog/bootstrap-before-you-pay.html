Bootstrap before you pay

How public profile creation fits a paid ASP without opening the whole database.

Every Metiora product package reads founder and startup memory. That means paid calls need real profile IDs. If only the operator API key could create those profiles, marketplace buyers would hit "not found" after payment. That is a terrible product experience and a support trap.

What bootstrap does

POST /v1/a2mcp/bootstrap is public and free. It accepts a small founder and startup payload, reuses an email if it already exists, and returns two UUIDs. Those IDs go straight into the paid body.

What it does not do

It does not unlock free package generation. It does not replace x402. It does not expose list or delete of other tenants. Ownership still requires the startup to belong to the founder on every paid run.

Why soft fail readiness

Early profiles are sparse. Hard rejecting "not investor ready" after a successful pay path feels like a broken product. Metiora still delivers packages and annotates low readiness so buyers leave with something useful and a clear next step to enrich memory.
