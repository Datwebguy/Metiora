Why Metiora locks internal routes

Two doors, two tickets. Payment for buyers. API key for operators.

Opening every route with no auth is convenient during a weekend hack. On a public domain it is a liability. Anyone can fill your Postgres with junk founders, generate packages for free, or poke simulated escrow helpers that were never meant to be marketplace payments.

Public surface

Health, avatar, marketing pages, services catalog, bootstrap, and paid A2MCP product paths. Buyers never need the Metiora API key. They need a working wallet flow for x402.

Private surface

Operator memory admin, internal package routes, and legacy OKX integration helpers. Those require a long random API key in production. Constant time compare reduces cheap timing tricks on the key check.

Secrets stay off GitHub

The public repo holds code and docs. Real keys live in Fly secrets and local ignored files. If a secret ever lands in git history, rotate it immediately and treat the old value as burned.
