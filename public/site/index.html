About

Built for the OKX agent marketplace

Metiora is a production Agent Service Provider, not a demo chatbot. It stores structured founder and startup memory, generates dual JSON and Markdown deliverables, and gates products with real x402 payments on X Layer.

01

Company memory first

Founder and startup profiles feed every package. Bootstrap creates IDs; paid services read that memory so outputs stay consistent across blueprints, investor packs, and health reviews.

02

x402 mainnet payments

Each product returns HTTP 402 without payment, then verifies and settles through the OKX facilitator using USDT0 on X Layer. Your payTo wallet receives marketplace fees.

03

Operator grade lockdown

Marketplace routes are public behind payment. Internal memory and admin routes require a Metiora API key so the open internet cannot write free packages into your database.

How to use

From bootstrap to paid delivery

Designed for agent buyers on OKX.AI and operators testing the live ASP.

1

Create profiles

Call POST /v1/a2mcp/bootstrap with founder email, full name, startup name, and industry. You receive founderProfileId and startupProfileId. No Metiora API key required.

2

Discover products

GET /v1/a2mcp/services lists prices, network, fee token, payTo, and full HTTPS endpoints. Use these exact URLs in marketplace registration.

3

Pay with x402

POST a product path without payment and you get 402. Complete the x402 exact payment on X Layer, then replay with payment headers. Settlement happens after successful generation.

4

Receive the package

Response includes structured JSON and Markdown content. Low readiness still delivers with a notice so buyers are not blocked after a successful pay path.

FAQ

Straight answers

Common questions from marketplace buyers and operators.

Do buyers need a Metiora API key?

No. Marketplace buyers use public A2MCP routes. Payment is the gate. The Metiora API key is only for your private operator routes such as internal memory admin tools.

Which chain and token?

X Layer mainnet, network id eip155:196. Fee token is official USDT0 at 0x779ded0c9e1022225f8e0630b35a9b54be713736. Do not register Polygon or other chain USDT addresses.

What body fields are required after payment?

founderProfileId and startupProfileId as UUIDs. Optional blueprintId for packages that can attach to a prior blueprint. Create IDs with the bootstrap endpoint first.

Where is the ASP listing?

Metiora is registered as OKX ASP agent #6860. Listing goes under review after activate. Keep endpoints online and returning 402 unpaid, then full delivery after pay.

Is the site the same as the API?

This website is the public product surface. The API runs behind the same domain for health, avatar, bootstrap, and paid services so marketplace URLs stay simple and stable.
