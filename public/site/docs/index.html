How Metiora works in production

Overview

Metiora is a headless Agent Service Provider for founders. Public marketplace routes live under /v1/a2mcp/* and charge with x402 on X Layer. Private operator routes need a Metiora API key.

Live base URL: https://agentmetiora.xyz

Buyer quickstart

  1. Create founder and startup profiles with bootstrap.
  2. List products from the services catalog.
  3. POST a product without payment to receive HTTP 402.
  4. Complete x402 payment with the OKX agent wallet flow.
  5. Replay the request with payment headers and receive JSON plus Markdown.
POST /v1/a2mcp/bootstrap
Content-Type: application/json

{
  "founder": { "email": "you@example.com", "fullName": "Ada Founder" },
  "startup": { "name": "Acme Labs", "industry": "AI" }
}

Public endpoints

  • GET /health · liveness and x402 status
  • GET /v1/a2mcp/services · catalog, prices, payTo, fee token
  • POST /v1/a2mcp/bootstrap · create profile IDs
  • POST /v1/a2mcp/tools · listable operations
  • POST /v1/a2mcp/startup-blueprint · 7 USDT
  • POST /v1/a2mcp/investor-ready · 7 USDT
  • POST /v1/a2mcp/grant-builder · 3 USDT
  • POST /v1/a2mcp/partnership-studio · 3 USDT
  • POST /v1/a2mcp/token-launch-kit · 3 USDT
  • POST /v1/a2mcp/startup-health · 2 USDT
  • GET /avatar.png · marketplace avatar

Do not list smoke test as a marketplace product. Keep it internal for payment path checks only.

Payments

  • Network: eip155:196 (X Layer mainnet)
  • Fee token: 0x779ded0c9e1022225f8e0630b35a9b54be713736 (USDT0)
  • Seller payTo: wallet set via Fly secret PAY_TO_ADDRESS
  • Scheme: x402 exact through the OKX facilitator SDK

Register the same token, network, and full HTTPS URLs that the live 402 response returns. Mismatched fee tokens are a common rejection reason.

Request body after payment

{
  "founderProfileId": "uuid",
  "startupProfileId": "uuid",
  "blueprintId": "uuid (optional)"
}

Create the UUIDs with bootstrap first. Ownership is enforced: the startup must belong to the founder. Generation soft fails on low readiness so buyers still receive a package with a readiness notice.

Operator notes

Production requires METIORA_API_KEY (min 16 chars), Postgres DATABASE_URL, and OKX facilitator credentials. Set secrets with Fly secrets, never in the git repo. 

fly secrets set METIORA_API_KEY=... PAY_TO_ADDRESS=0x... \
  OKX_API_KEY=... OKX_SECRET_KEY=... OKX_PASSPHRASE=...

Protected routes accept x-api-key or Authorization: Bearer. CORS defaults to empty origins in production for server to server and agent clients.

Marketplace listing checklist

  • Avatar: https://agentmetiora.xyz/avatar.png
  • All six product URLs on agentmetiora.xyz (not localhost)
  • Prices match catalog ($7 / $7 / $3 / $3 / $3 / $2)
  • Unpaid POST returns 402, not 404 or hang
  • Required body fields published in listing metadata
  • Agent stays online during OKX review (Fly min machines running)

Security boundaries

  • Marketplace paths: public, payment gated
  • Internal memory and legacy /okx routes: API key required
  • Never commit .env, API keys, or facilitator secrets
  • Metrics stay private unless METRICS_PUBLIC=true
